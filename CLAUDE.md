# varius 저장소 공통 가이드

> 이 저장소는 **여러 잡다한 프로젝트를 모아놓은 모노레포**다.
> 부모 디렉토리의 [`~/claude-works/CLAUDE.md`](../CLAUDE.md)는 PHP/CodeIgniter 4 백엔드 프로젝트를
> 전제로 하지만, 이 저장소는 그 전제를 따르지 않는다. **아래 규칙이 이 저장소 전체에서 우선한다.**

## 저장소 구조

- 프로젝트마다 **루트에 서브디렉토리**를 만들어 격리한다. 예: `rtic/`(realtime intercom project).
- 각 서브디렉토리는 **자체 `CLAUDE.md`·`README.md`를 반드시 보유**한다.
  - `CLAUDE.md`: 해당 프로젝트의 스택·아키텍처·개발/검증 명령어 등 Claude Code 작업 규칙.
  - `README.md`: 사람이 읽는 프로젝트 개요·실행 방법.
- 새 프로젝트를 시작할 때는 이 두 파일부터 만든다.

## 이슈 라벨 = 서브디렉토리

- GitHub 이슈를 작성할 때는 **대상 서브디렉토리명과 동일한 라벨**을 붙인다. 예: `rtic/` 관련 이슈는 `rtic` 라벨.
- 이슈 작업(`dev-issue`/`start-issue` 등)에 착수할 때는 이슈에 붙은 라벨을 **작업 대상 디렉토리**로 삼는다 —
  해당 서브디렉토리 범위 밖의 파일은 그 이슈 작업에서 건드리지 않는다.
- 아직 해당 라벨이 저장소에 없다면 이슈 생성 전에 `gh label create <서브디렉토리명>`으로 먼저 만든다.
- 여러 프로젝트에 걸친 이슈이거나 저장소 공통(루트 `CLAUDE.md`/`README.md` 등) 변경이면 라벨을 생략할 수 있다.

## 사용 언어

- 프로젝트별로 **사용 언어가 다를 수 있다** (PHP, Node.js, Python, Go 등). 전역 PHP 전제를 이 저장소에 강제하지 않는다.
- 각 프로젝트의 언어·프레임워크·코드 스타일 규칙은 해당 프로젝트의 `CLAUDE.md`가 정의한다.

## Git 워크플로우

부모 저장소들과 달리 이 저장소는 **PR 없이 직접 머지**한다.

```
feature/* → dev → main
```

- `feature/*` 작업 완료 후 **PR을 생성하지 않고** `dev`로 직접 머지한다.
- `dev`가 배포 가능한 상태가 되면 `dev → main`으로 직접 머지한다.
- 브랜치 전략(`feature → dev → main`) 자체는 유지하되, 리뷰 절차(PR)만 생략한다.
- 커밋 메시지 규칙은 전역 규칙([`~/.claude/rules/git-workflow.md`](~/.claude/rules/git-workflow.md))의
  **이모지 + Conventional Commits + 한국어 설명**을 그대로 따른다.

## CI/CD

- 이 저장소는 **CI/CD 파이프라인을 두지 않는다.**
- 모든 검증(테스트·정적 분석·빌드 확인 등)은 **로컬에서** 수행한 뒤 머지한다.
- 각 프로젝트의 구체적인 로컬 검증 명령은 해당 프로젝트 `CLAUDE.md`에 명시한다.

## 문서 전용 변경

- `*.md` 문서만 수정하는 경우 전역 규칙의 "문서 전용 예외"를 그대로 적용한다: `dev`에 직접 커밋.
- 코드가 한 줄이라도 섞이면 위 Git 워크플로우(`feature/*` → `dev`)를 따른다.
