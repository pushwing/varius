# 지도 목록 메뉴(월별/일별) 사이드 패널 설계

## 배경

현재 `/map` 페이지(`app/Views/map.php`)는 Leaflet 지도 위에 날짜별 마커·경로선만 그려주고, 특정
날짜의 위치로 바로 이동하려면 지도를 직접 스크롤/줌해서 찾아야 한다. 우상단에는 날짜별 색상
범례(`#legend`)만 있고 탐색 기능은 없다.

이 설계는 좌측에 **월별 → 일별 목록 메뉴**를 상시 노출하는 사이드 패널을 추가해, 사용자가
목록에서 날짜를 클릭하면 지도가 해당 날짜의 위치로 이동하고 사진 레이어가 자동으로 열리도록 한다.

## 범위

- **프론트엔드(뷰)만 변경한다.** 백엔드(`RouteController`, `RouteVisualizationService`, `/routes`
  JSON API)는 변경하지 않는다 — `/routes` 응답에 이미 날짜별 `points`(경로 좌표 전체)와
  `clusters`(같은 장소 사진 묶음)가 포함되어 있어, 월별/일별 그룹핑은 클라이언트에서 충분히
  구성 가능하다.
- 변경 파일: `app/Views/map.php` (인라인 `<style>`/`<script>` 확장). 새 파일을 추가하지 않는다 —
  이 프로젝트는 빌드 스텝 없이 뷰 하나에 스타일·스크립트를 인라인으로 두는 기존 패턴을 따른다.

## UI 레이아웃

- 우상단 `#legend`(날짜별 색상 범례)는 **제거**하고 새 사이드 패널로 완전히 대체한다.
- 좌측에 상시 표시되는 사이드 패널 `#route-sidebar`를 신설한다: 폭 약 280px, 흰 배경 +
  box-shadow, 세로 스크롤(`overflow-y: auto`), 전체 높이(100vh).
- `#map`은 사이드 패널 폭만큼 우측으로 밀려 나머지 공간을 채운다(`left: 280px`로 조정, 기존
  `inset: 0` 대신).
- 데이터가 없는 경우(기존 `#empty` 상태) 사이드 패널도 빈 상태로 둔다(목록 항목 없음).

## 목록 구조·정렬

- 2단 트리: **월(YYYY년 M월) → 그 달의 일자(M월 D일 · N장)**.
- **정렬은 최신 우선(역순)**: 가장 최근 월이 맨 위, 그 달 안에서도 가장 최근 일자가 맨 위.
  (여행/사진 목록은 최근 기록부터 확인하는 흐름이 자연스럽기 때문.)
- 데이터 소스: `/routes` 응답의 `dates` 배열은 이미 `taken_at` 오름차순으로 정렬돼 내려온다
  (`RouteVisualizationService` 문서화된 동작). 클라이언트에서 월별로 묶은 뒤 표시 시점에만
  월 배열과 각 월의 일자 배열을 역순으로 뒤집어 렌더링한다. 원본 배열 순서는 그대로 두어
  `clusterRegistry` 인덱스 매핑(기존 "더보기" 팝업 로직)과 충돌하지 않게 한다.

## 인터랙션

- **월 헤더 클릭**: 그 달의 일자 목록 `hidden` 속성을 토글한다(펼침 ↔ 접힘). **다른 월의 펼침
  상태에는 영향을 주지 않는다** — 여러 월을 동시에 펼쳐볼 수 있는 단순 토글이다.
- **기본 펼침 상태**: 페이지 로드 시 가장 최근 월만 펼쳐진 상태, 나머지 월은 접힌 상태로
  시작한다.
- **일자 항목 클릭**:
  1. 그 날짜의 전체 좌표(`group.points`의 `[lat, lng]` 배열)로 `map.fitBounds(...,
     { padding: [40, 40] })` 호출 — 그날 방문한 모든 장소가 보이도록 지도를 이동/줌한다.
  2. 그 날짜의 **첫 번째 클러스터**(`group.clusters[0]`)에 대응하는 `clusterRegistry` 인덱스로
     기존 `openLayer(clusterIndex)`를 호출해 사진 레이어(`#photo-layer`)를 연다. (여러 클러스터가
     있어도 첫 번째 클러스터 사진만 보여준다 — 나머지 클러스터는 지도 위 마커를 직접 클릭해
     "더보기"로 열람 가능한 기존 경로를 그대로 유지한다.)
  3. 클릭한 일자 항목에 활성 표시(배경색 강조 클래스)를 부여하고, 다른 일자 항목의 활성
     표시는 해제한다(한 번에 하나만 활성).

## 구현 메모

- `render(dates)` 함수 내에서 기존 마커/경로선/`clusterRegistry` 구성 로직은 그대로 두고,
  같은 순회 과정에서 사이드 패널 렌더링에 필요한 모델(`monthGroups`: 월별로 묶인
  `{date, pointsLatLngs, count, firstClusterIndex}` 목록)을 함께 구성한다.
- 사이드 패널 DOM은 `render()` 마지막에 한 번 생성한다(현재 페이지는 로드 시 한 번만
  `/routes`를 fetch하므로 재렌더링 로직은 필요 없다).
- 이벤트 바인딩은 기존 "더보기" 팝업과 동일하게 이벤트 위임(`document.body`에 클릭 리스너,
  `closest()`로 대상 판별) 방식을 따른다 — 사이드 패널 항목도 매 렌더링 시 새로 그려지는
  DOM이므로 개별 리스너 등록 대신 위임이 일관적이다.

## 검증

- 이 변경은 뷰의 인라인 JS만 수정하므로 PHP 클래스 로직 변경이 없어 PHPUnit 테스트 대상이
  아니다.
- 프로젝트 규칙상 런타임 표면(지도 페이지) 변경은 테스트만으로 끝내지 않고 실제 브라우저
  구동으로 확인한다: `/map` 접속 → 사이드 패널 렌더링 확인 → 월 토글 확인 → 일자 클릭 시
  지도 이동 + 사진 레이어 오픈 확인.
- `composer ci`(CS Fixer → PHPStan → PHPUnit)를 회귀 확인 차원에서 실행한다.
